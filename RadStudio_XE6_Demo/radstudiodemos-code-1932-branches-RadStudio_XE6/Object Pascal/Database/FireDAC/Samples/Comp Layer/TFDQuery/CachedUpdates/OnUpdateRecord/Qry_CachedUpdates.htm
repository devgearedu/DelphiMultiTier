1         Cached Updates

The demo demonstrates caching of the user changes in the case of read-only dataset and applying them to database using OnUpdateRecord event of TADQuery.

The qryProducts component retrieves read-only dataset from RDBMS:

select p.*, c.CategoryName from {id Products} p, {id Categories} c  where p.CategoryID = c.CategoryID

 

In the RDBMS it has on-to-many relation between Categories and Products tables.

For applying the changes of this dataset they need to be used TADUpdateSQL components (usProducts and usCategories). First one is used for ‘Products’ table; second one for ‘Categories’.

 

1. Applying updates

  qryProducts.ApplyUpdates;

 

After calling method ApplyUpdates of qryProducts it’s called OnUpdateRecord event of qryProducts.

On this event we try applying the changes in the specific order. usProducts applies changes into Products table in RDBMS, usCategories into Categories one.

In the beginning we apply changes into Products (detail) table, second into Categories (master) one.

 

procedure TfrmCachedUpdates.qryProductsUpdateRecord(ASender: TDataSet;

  ARequest: TADPhysUpdateRequest; var AAction: TADErrorAction;

  AOptions: TADPhysUpdateRowOptions);

begin

  if not (ARequest in [arLock, arUnlock]) then begin

    usProducts.ConnectionName := qryProducts.ConnectionName;

    // set DataSet property to TADQuery component that has the opened dataset

    usProducts.DataSet := qryProducts;

    // first, apply user changes into Products table

    usProducts.Apply(ARequest, AAction, AOptions);

 

    if AAction = eaApplied then begin

      usCategories.ConnectionName := qryProducts.ConnectionName;

      // set up property as well as in usProducts

      usCategories.DataSet := qryProducts;

      // second, apply user changes into Categories table

      usCategories.Apply(ARequest, AAction, AOptions);

    end;

  end;

  AAction := eaApplied;

end;

 

2. Canceling updates

  qryProducts.CancelUpdates;

 

3. Committing updates

  qryProducts.CommitUpdates;

 

4. Reverting record

  qryProducts.RevertRecord;

 

5 . Undoing last change

  qryProducts.UndoLastChange(True);
