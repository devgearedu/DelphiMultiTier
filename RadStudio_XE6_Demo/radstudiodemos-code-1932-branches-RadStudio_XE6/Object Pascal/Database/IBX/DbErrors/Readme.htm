Description This example demonstrates:

  • Trapping and Controlling Database Errors
Database Error Trapping:
Delphi displays errors that arise from user interaction with database tables as application level exceptions. The new OnPostError, OnEditError and OnDeleteError events of the TIBTable objects allow you to trap these errors closer to their origin.

You may encounter several different database engine errors: key violations, missing required fields, referential integrity errors, incorrect foreign keys and so on.

This example requires that Interbase XE be installed. Make sure the server is running before you run the example.
